# 01 程序入口、变量、函数

## 🎯 这一章解决什么问题

这一章先解决 Go 最基础的 3 个问题：

1. 程序为什么从 `package main` 和 `func main()` 开始
2. 变量为什么既有 `var`，又有 `:=`
3. 函数为什么既可以只做事，也可以返回多个值

如果这一章通了，后面你看 Go 代码就不会一直停在“这个语法是什么意思”。

## 🧩 最小可运行代码

```go
package main

import "fmt"

func add(a int, b int) int {
    return a + b
}

func main() {
    name := "Go"
    result := add(2, 3)

    fmt.Println("hello", name)
    fmt.Println("2 + 3 =", result)
}
```

## 🔍 逐行解释

### `package main`

这句不是类声明，而是 **包声明**。

在 Go 里：

- 文件先属于某个 package
- package 是组织代码的单位
- `main` 是一个特殊包
- 只有 `package main` 才能编译成可执行程序

这和 Java 很不一样。

Java 通常是：

- 先定义类
- 再在类里定义 `public static void main`

Go 则是：

- 先定义包
- 再在包里定义 `func main()`

### `import "fmt"`

导入标准库 `fmt`。

你可以先把它理解成“格式化输出包”。

常见用途：

- `fmt.Println()`：输出一行
- `fmt.Printf()`：按格式输出
- `fmt.Sprintf()`：生成格式化字符串

### `func add(a int, b int) int`

这是一个函数定义。

含义：

- 函数名：`add`
- 参数：`a int, b int`
- 返回值：`int`

Go 的参数类型写在变量名后面，这一点和 Java 不同。

Java：

```java
int add(int a, int b)
```

Go：

```go
func add(a int, b int) int
```

### `func main()`

这是程序入口函数。

当你执行：

```bash
go run hello.go
```

或：

```bash
go build
```

Go 会从 `main()` 开始执行。

### `name := "Go"`

`:=` 是 **短变量声明**。

它表示：

- 声明一个新变量
- 同时完成赋值
- 类型由 Go 自动推断

所以这句等价于：

```go
var name string = "Go"
```

### `result := add(2, 3)`

意思是：

- 调用 `add(2, 3)`
- 拿到返回值
- 定义新变量 `result`

如果你来自 Java，可以把它类比成：

```java
int result = add(2, 3);
```

## 🆚 Java 开发怎么理解

先用下面这张映射表记忆：

```text
Java class Main + main()   ≈ Go package main + func main()
Java import                ≈ Go import
Java 方法                  ≈ Go func
Java 局部变量声明          ≈ Go var / :=
```

但要注意，Go 不以 class 为中心，而是以 **package + func + struct** 为中心。

## ✅ 有返回值函数 vs 无返回值函数

### 有返回值函数

```go
func add(a int, b int) int {
    return a + b
}
```

这种函数可以拿来参与表达式：

```go
result := add(2, 3)
fmt.Println(add(2, 3))
```

### 无返回值函数

```go
func printHello() {
    fmt.Println("hello")
}
```

这种函数只负责“做动作”，不能当成值用：

```go
printHello()          // ✅
fmt.Println(printHello()) // ❌
```

这就是你之前遇到的：

```text
xxx() (no value) used as value
```

## ⚠️ 注意点 / 易错点

### 1. `:=` 不是普通赋值

`:=` 的含义是“声明新变量并赋值”，不是简单的“重新设置值”。

```go
x := 10
x = 20   // 给已有变量赋值
x := 30  // ❌ 同一作用域下重复声明
```

### 2. `:=` 只能在函数内部使用

下面是错的：

```go
package main

name := "Go" // ❌
```

包级变量要用 `var`：

```go
var name = "Go"
```

### 3. `main()` 只能有一个入口版本

同一个目录、同一个 `package main` 下，不能有多个 `func main()`。

否则会报：

```text
main redeclared in this block
```

### 4. `fmt.Printf` 和 `fmt.Println` 不一样

- `fmt.Println(a, b)`：直接输出
- `fmt.Printf("%d", n)`：需要格式化字符串

所以这类写法通常不对：

```go
fmt.Printf(add(2, 3))
```

更合理的是：

```go
fmt.Println(add(2, 3))
fmt.Printf("%d\n", add(2, 3))
```

### 5. IDE 报错不一定是语法错

你之前遇到过：

- 中文目录名导致包路径非法
- 同目录多个 `main()` 导致整个目录编译失败

所以看到红线时，先区分：

- 是单文件写法错了
- 还是目录结构、包路径、编译范围出问题

## 📌 本章小结

这一章你要记住 5 句话：

1. `package main` 表示可执行入口包
2. `func main()` 是程序入口函数
3. `func` 用来定义函数
4. `var` 和 `:=` 都能声明变量，但 `:=` 更常用于函数内部
5. 无返回值函数不能拿来当值使用
