# 06 包、模块、工程组织

## 🎯 这一章解决什么问题

这章不是讲“语法”，而是讲 Go 工程到底怎么被组织起来。

需要解决：

1. package 和 module 到底是什么关系
2. `go.mod` 在项目里扮演什么角色
3. 为什么目录名会影响包路径
4. `go run file.go`、`go run .`、`go build .` 到底差在哪

## 1️⃣ package：目录级组织单位

Go 把目录当成包的主要承载单元。

通常一个目录就是一个 package。

例如：

```text
examples/hello/
  main.go
```

这个目录里的 Go 文件通常都应该写同一个 package 名。

## 2️⃣ module：整个项目的依赖单位

`go.mod` 里定义了当前项目的模块名：

```go
module github.com/xrjjing/Learn4Go
```

你可以把 module 理解成：

- 这个仓库的根标识
- 依赖管理边界
- 包导入路径的前缀来源

## 3️⃣ package 路径从哪里来

比如模块是：

```text
github.com/xrjjing/Learn4Go
```

那目录：

```text
examples/hello
```

对应的包路径大致就是：

```text
github.com/xrjjing/Learn4Go/examples/hello
```

所以目录名是否合法，会直接影响包路径是否合法。

## ⚠️ 为什么中文目录名会出问题

你前面已经踩过这个坑。

类似：

```text
01.Go语言基础
```

被放进包路径时，可能导致：

```text
malformed import path ... invalid char '语'
```

单文件 `go run hello.go` 可能还能过，但按包/目录构建时 IDE 和 gopls 往往会报错。

所以 **教学目录可以中文，真正需要作为 Go 包编译的目录尽量用纯 ASCII。**

## 4️⃣ `go run` 和 `go build` 的编译边界

### 按文件运行

```bash
go run hello.go
```

只按你指定的文件编译。

### 按目录运行

```bash
go run .
```

会把当前目录下属于同一包的 `.go` 文件一起编译。

### 按文件构建

```bash
go build -o hello hello.go
```

只编译这个文件。

### 按目录构建

```bash
go build -o hello .
```

编译当前目录整个包。

## 🔥 为什么同目录多个 `main()` 会报错

如果一个目录下有：

- `hello.go`
- `for_time.go`

并且两者都是：

```go
package main
func main() {}
```

那按目录编译时就会报：

```text
main redeclared in this block
```

因为 Go 认为它们都属于同一个包，入口函数重复了。

## 🆚 Java 开发怎么理解

- Java 更强调包名和类路径
- Go 更强调目录和模块
- Java 常见是 `src/main/java/...`
- Go 更常见是模块根目录直接承载源码子目录

## ✅ 常见目录组织建议

对学习项目来说，先记这个就够：

```text
cmd/         # 可执行程序入口
internal/    # 项目内部包
examples/    # 独立示例
docs/        # 学习文档
```

## ⚠️ 注意点 / 易错点

### 1. 一个目录内的 `.go` 文件通常必须同包名

同目录混用不同 package，通常会直接编译失败。

### 2. 目录名不是“随便起”

特别是要按包编译时，尽量使用：

- 小写字母
- 数字
- `-` 或 `_`

### 3. demo 最好单独目录隔离

不要在同一个 `demo/` 目录里放多个互相独立的 `main()` 程序。

更推荐：

```text
demo/hello/main.go
demo/for-time/main.go
```

### 4. `go.mod` 在模块根目录

绝大多数命令都默认以模块根目录为上下文，更稳。

### 5. gopls 报错经常是结构问题，不是语法问题

比如：

- 非法目录名
- 多个 `main()`
- 包声明不一致

## ▶️ 本章建议运行命令

```bash
go run ./examples/packages
go env GOMOD GOPATH GOWORK
```

也建议自己试：

```bash
go run hello.go
go build -o hello hello.go
go run .
go build -o app .
```

感受“按文件”和“按目录”的差异。

## 📌 本章小结

你要记住：

1. package 是代码组织单位，module 是项目依赖边界
2. `go.mod` 定义模块根身份
3. 目录名会影响包路径
4. `go run file.go` 和 `go run .` 编译范围不同
5. 同目录多个 `main()` 会冲突
