# 02 流程控制、作用域、短变量声明

## 🎯 这一章解决什么问题

这一章解决 4 个非常高频的问题：

1. Go 里的 `if / for / switch` 到底怎么写
2. 为什么 Go 没有 `while`
3. 为什么 `:=` 有时候复用旧变量，有时候又像新建变量
4. `err` 被遮蔽到底是什么意思

这章如果没吃透，后面你会在 `if err != nil`、循环和作用域上不停踩坑。

## 🧩 最小可运行代码

```go
package main

import "fmt"

func main() {
for i := 1; i <= 3; i++ {
    if i%2 == 0 {
        fmt.Println(i, "是偶数")
    } else {
        fmt.Println(i, "是奇数")
    }
}

score := 85
switch {
case score >= 90:
    fmt.Println("A")
case score >= 80:
    fmt.Println("B")
default:
    fmt.Println("C")
}
}
```

## 🔍 Go 的流程控制长什么样

### `if`

Go 的 `if` 不需要括号：

```go
if x > 0 {
fmt.Println("positive")
}
```

还支持“先声明、再判断”：

```go
if n := len(name); n > 0 {
fmt.Println(n)
}
```

这里的 `n` 只在这个 `if` 块里有效。

### `for`

Go 没有单独的 `while`，只有 `for`。

但 `for` 可以写成 3 种形态：

```go
for i := 0; i < 3; i++ {}
for x < 10 {}
for {}
```

所以：

- `for i := ...` 像传统 for
- `for x < 10` 就是 Go 的 while
- `for {}` 就是无限循环

### `switch`

Go 的 `switch` 比 Java 更灵活：

```go
switch day {
case 1:
fmt.Println("Mon")
case 2:
fmt.Println("Tue")
default:
fmt.Println("Unknown")
}
```

还可以不写表达式：

```go
switch {
case score >= 90:
fmt.Println("A")
case score >= 80:
fmt.Println("B")
}
```

这种写法非常适合写区间判断。

## 🔒 作用域：变量到底活到哪里

Go 的作用域规则很朴素：

- `{}` 内声明的变量，只在这个块里有效
- 内层可以定义和外层同名的变量
- 一旦定义同名变量，外层变量会被遮住（shadowing）

示例：

```go
x := 10

if true {
x := 20
fmt.Println(x) // 20
}

fmt.Println(x) // 10
```

这里不是把外面的 `x` 改了，而是在 if 里面新建了一个 `x`。

## ✅ `:=` 为什么有时“复用”，有时“新建”

关键规则只有一句：

**`:=` 在当前作用域里至少要有一个新变量；已存在的同名变量会复用，但只限当前作用域。**

### 场景 1：同一作用域里复用旧变量

```go
file, err := openA()
data, err := readB()
```

第二句里：

- `data` 是新变量
- `err` 是当前作用域里已经存在的变量
- 所以 `err` 被复用

### 场景 2：新作用域里创建同名变量

```go
file, err := openA()
if err != nil {
return
}

if true {
data, err := readB()
_ = data
_ = err
}
```

这里 `if` 内部的 `err` 是一个新的变量，不是外面的 `err`。

这就是“变量遮蔽”。

## 🆚 Java 开发怎么理解

Java 里也有块级作用域，但你在 Java 里很少会用短变量声明，所以对“同名变量遮蔽”的体感没 Go 这么强。

Go 里因为 `if err := ...; err != nil {}`、`x, err := ...` 非常常见，作用域和遮蔽就成了日常问题。

## ⚠️ 注意点 / 易错点

### 1. `if err := ...; err != nil {}` 里的 `err` 只活在 if 里

```go
if err := do(); err != nil {
fmt.Println(err)
}

// 这里不能再用 err
```

### 2. 不要把遮蔽误解成赋值

```go
x := 1
{
x := 2
}
```

外面的 `x` 没变。

### 3. `switch` 默认不会自动 fallthrough

和 Java 不同，Go 的 `switch` 每个 case 默认执行完就结束。

如果你确实要继续落到下一个 case，要显式写：

```go
fallthrough
```

但基础阶段尽量少用。

### 4. `for range` 后面会频繁出现值拷贝问题

这章先知道：`range` 遍历时拿到的是值，不一定是原对象本身。后面讲 slice、map、struct 时再深入。

### 5. 不要过早把 Go 的 `for` 想成 Java 增强 for 的翻版

Go 的 `for range` 语义和 Java `for-each` 有相似点，但细节差很多，尤其是：

- 下标和值同时返回
- 值可能是副本
- 遍历 map 时无序

## 📌 本章小结

这章核心只有 4 句话：

1. Go 用一个 `for` 统一了 for / while / infinite loop
2. Go 的 `if` 和 `switch` 都可以很简洁
3. `:=` 在当前作用域里至少要引入一个新变量
4. 同名变量在内层作用域里会遮蔽外层变量
